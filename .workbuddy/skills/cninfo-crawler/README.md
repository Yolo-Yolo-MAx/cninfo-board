# 巨潮资讯网公告爬虫 + 飞书写入 使用说明

## 功能概述

每日自动从巨潮资讯网爬取含有关键词（破产/重整/拍卖/法拍/司法拍卖）的公告，补充申万三级行业与公司简介后写入飞书多维表格。

---

## 整体流程

```
巨潮爬虫搜索公告
      │
      ▼
过滤：只保留 [重整] 或 [拍卖] 类型
      │
      ▼
去重：同一股票+同一标题+同一日期只保留一条
      │
      ▼
读取飞书已有记录，排除已写入的公告
      │
      ▼
对每条新公告：
  ├─ 调用 iFinD MCP 查询 申万一级/二级/三级行业
  └─ 调用 iFinD MCP 查询 公司简介（fallback 到主营业务）
      │
      ▼
批量写入飞书多维表格
```

---

## 文件说明

| 文件 | 用途 |
|------|------|
| `cninfo_crawler_v2.py` | 主爬虫：爬取公告 + 查询行业/简介 + 写入飞书 |
| `backfill_industry.py` | 回填脚本：更新飞书已有记录的行业和简介 |
| `call.py` | iFinD MCP 调用封装（由 skill 安装时提供） |
| `mcp_config.json` | iFinD 鉴权 token 配置 |

---

## 飞书表格字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 股票代码 | 单行文本 | 如 `000536` |
| 公司名称 | 单行文本 | 如 `华映科技` |
| 公告日期 | 日期 | 毫秒时间戳 / 1000 |
| 公告标题 | 单行文本 | 原始标题（已去除 `<em>` 高亮标签） |
| 公告链接 | 超链接 | 跳转到巨潮公告详情页 |
| 公告类型 | 单选 | `重整` 或 `拍卖` |
| 所属行业 | 多选 | 申万三级，如 `电子-光学光电子-面板` |
| 公司简介 | 多行文本 | 最多 500 字 |

## 核心逻辑说明

### 1. 公告类型判断（`get_ann_type`）

```python
zz_keywords = ["重整", "破产重整", "重整计划", "重整程序", "重整受理",
               "重整宣告", "申请重整", "重整投资人"]
pm_keywords = ["拍卖", "法拍", "司法拍卖", "资产拍卖", "股权拍卖",
               "网络拍卖", "强制拍卖", "变卖"]

# 规则：
#   同时含重整关键词 + 拍卖关键词 → "拍卖"（拍卖优先）
#   只含重整关键词               → "重整"
#   其他                         → None（跳过）
```

### 2. 股票代码格式化（`fmt_code`）

将纯数字代码转为 iFinD 需要的带交易所后缀格式：

| 输入 | 输出 | 规则 |
|------|------|------|
| `000536` | `000536.SZ` | 000/001/002/003/300/301 开头 → `.SZ` |
| `600519` | `600519.SH` | 600/601/603/605/688 开头 → `.SH` |
| `83XXXX` | `83XXXX.BJ` | 8 或 4 开头 → `.BJ` |

### 3. 申万三级行业获取（`get_industry_and_brief`）

iFinD 的 `get_stock_info` 接口不支持一次返回三级行业的完整路径，需要分别查询再拼接：

```python
# 三个独立查询
row1 = get_ifind_field(code_fmt, "申万一级行业")   # → "电子"
row2 = get_ifind_field(code_fmt, "申万二级行业")   # → "光学光电子"
row3 = get_ifind_field(code_fmt, "申万三级行业")   # → "面板"

# 去掉罗马数字（Ⅱ、Ⅲ 等），再拼接
industry = "电子-光学光电子-面板"
```

**关键坑**：查询词不能带括号，即必须用 `"申万一级行业"` 而不是 `"所属申万行业（一级）"`，后者 iFinD 无法识别。

### 4. iFinD 查询重试机制（`get_ifind_field`）

iFinD MCP 偶尔返回空结果（HTTP 200 但 answer 为空），需要重试：

```python
def get_ifind_field(code_fmt, query_text, retries=2):
    for attempt in range(retries + 1):
        result = ifind_call("stock", "get_stock_info", {...})
        if result.get("ok"):
            row = parse_md_table(answer)
            if row:
                return row
        time.sleep(0.5)   # 失败后等待重试
    return {}
```

### 5. 公司简介 fallback

部分股票 iFinD 没有"公司简介"字段，此时 fallback 到"主营业务"：

```python
brief = row_brief.get("公司简介", "")
if not brief:
    brief = row_biz.get("主营业务", "")
```

### 6. Markdown 表格解析（`parse_md_table`）

iFinD 返回的数据是 markdown 格式表格，需要手动解析：

```markdown
|证券代码|证券简称|所属申万行业|
|---|---|---|
|000536.SZ|华映科技|面板|
```

解析要点：
- 跳过含 `---` 的分隔行
- 第一行是表头，第二行是数据
- 按位置对齐（不要用字段名去 dict 里取，因为表头/数据列数可能不一致）

## 回填脚本 `backfill_industry.py`

用于修正飞书已有记录（例如：之前行业只有一级，需要补充为三级）。

**与爬虫的区别**：
- 读取飞书**所有**记录（不限日期）
- 按股票代码去重（每只股票只查一次 iFinD）
- **强制覆盖**已有字段（不设 `not fields.get("xxx")` 判断）
- 逐条调用飞书 PUT 接口更新（飞书不支持 batch update）

## 定时运行

爬虫设计为每次运行处理「昨天 09:00 → 今天 09:00」的公告，适合每天定时执行：

```bash

```

## 常见问题

### Q: 行业显示不正确（只有一级或二级）？
A: iFinD 查询偶尔失败，确保 `get_ifind_field` 有重试机制（retries=2）。可手动运行 `backfill_industry.py` 重新回填。

### Q: 公司简介为空？
A: 该股票 iFinD 可能没有"公司简介"数据，代码会自动 fallback 到"主营业务"。如果两者都为空则是 iFinD 数据缺失，非代码 bug。

### Q: 飞书写入失败？
A: 检查 `FEISHU_APP_ID` / `FEISHU_APP_SECRET` 是否正确，以及多维表格的字段名称是否与代码中的完全一致（区分单选/多选类型）。
