# 重整与拍卖公告看板 - 功能说明

## 项目简介

这是一个**自动化的A股重整与拍卖公告追踪系统**，实时抓取巨潮资讯网的上市公司公告，自动分类整理后通过网页看板、飞书表格、飞书通知三种方式呈现给用户。

---

## 核心功能

### 1. 自动化公告爬取

**数据来源：** 巨潮资讯网（cninfo.com.cn）- 中国证监会指定信息披露网站

**爬取范围：**
- **重整公告：** 上市公司破产重整、司法重整相关公告
- **拍卖公告：** 上市公司股份司法拍卖、资产拍卖相关公告
- **不包含：** 破产清算类公告（已过滤）

**爬取频率：** 每日3次自动更新
- **12:00** — 抓取 00:00~12:00 时段的公告
- **18:00** — 抓取 12:00~18:00 时段的公告
- **24:00** — 抓取 18:00~24:00 时段的公告

**技术实现：**
- Python 爬虫脚本 `cninfo_crawler_v2.py`
- Windows 任务计划程序定时触发
- 自动检测网络代理并适配

---

### 2. 行业信息自动补全

**数据来源：** 同花顺 iFinD 金融数据终端（MCP接口）

**补全字段：**
- **申万三级行业分类**（如：商贸零售-贸易Ⅱ-贸易Ⅲ）
- **公司主营业务简介**（从 iFinD 自动提取）

**行业分类兜底策略：**
1. 优先使用申万行业分类（一级→二级→三级完整拼接）
2. 申万数据缺失时，自动切换为同花顺行业分类
3. 仍缺失时，从二级行业名推导一级行业

---

### 3. 飞书多维表格同步

**功能：** 将爬取结果自动写入飞书多维表格，方便团队协作和数据分析

**表格字段：**
- 股票代码（code）
- 公司名称（name）
- 行业分类（industry）- 多选字段
- 公告类型（announcement）- 重整/拍卖
- 公告标题（title）
- 公告链接（link）
- 公告日期（date）
- 发布时间戳（publish_time）
- 更新时段（update_slot）- 12/18/24

**写入策略：**
- 自动去重（基于公告ID）
- 按更新时段分批次写入
- 支持历史数据回溯

---

### 4. 飞书机器人通知

**触发时机：** 每次爬虫任务完成后自动发送

**通知内容：**
- ✅ 新增公告数量
- 📊 表格累计公告总数
- 📋 最新公告列表（前5条，含标题和链接）

**无新公告时：** 发送"没有发现新的重整/拍卖公告"提示

---

### 5. 网页看板（前端展示）

**访问地址：**
- GitHub Pages：`https://yolo-yolo-max.github.io/cninfo-board/`
- Cloudflare Pages：（待配置）

**核心功能：**

#### 5.1 统计面板
- **累计公告数量** — 历史上所有爬取到的公告总数
- **今日新增数量** — 当天新抓取的公告数量
- **重整公告数量** — 重整类公告总数
- **拍卖公告数量** — 拍卖类公告总数

#### 5.2 筛选功能
- **按类型筛选：** 全部 / 重整 / 拍卖
- **搜索功能：** 支持按股票代码、公司名称、公告标题模糊搜索
- **实时过滤：** 输入关键词后立即显示匹配结果

#### 5.3 公告卡片展示
每条公告以卡片形式展示：
- **公告类型标签** — 重整（橙色）/ 拍卖（蓝色）
- **公告日期**
- **公告标题** — 点击可跳转巨潮资讯网原文
- **公司股票代码** — 等宽字体显示
- **公司名称**
- **申万三级行业分类** — 如"纺织服饰-饰品-钟表珠宝"
- **公司主营业务简介** — 2行省略显示

#### 5.4 Excel 导出功能
- **今日新增下载** — 导出当天新抓取的公告（Excel格式）
- **全部记录下载** — 导出历史上所有公告（Excel格式）

#### 5.5 响应式设计
- 支持桌面端和移动端访问
- 移动端自动调整布局

**设计风格：**
- 金融风格配色（藏青色 + 金色）
- 卡片式布局
- 简洁专业

---

### 6. 数据接口（JSON）

**文件位置：**
- `data/announcements_all.json` — 全部公告数据
- `data/announcements_today.json` — 今日新增公告数据

**数据用途：**
- 供网页看板读取和展示
- 可供其他系统调用（API接口）
- 支持数据备份和历史查询

---

### 7. 自动 Git 同步

**功能：** 每次爬虫任务完成后，自动将更新后的数据推送到 Git 仓库

**推送目标：**
- **GitHub 仓库：** `Yolo-Yolo-MAx/cninfo-board`（公开）
- **Gitee 仓库：** `yolo-yo/cninfo-board`（国内镜像）

**推送内容：**
- 更新后的 `announcements_all.json`
- 更新后的 `announcements_today.json`
- 更新后的 `index.html`（如果有修改）

**自动化流程：**
1. 爬虫执行完成
2. 更新本地 JSON 数据文件
3. Git commit（提交信息：自动更新公告数据）
4. Git push 到 GitHub + Gitee

**代理适配：**
- 自动检测系统代理设置
- 支持公司网络环境

    ↓

## 注意事项

1. **公告类型限制：** 仅抓取"重整"和"拍卖"类公告，不包含"破产清算"
2. **数据准确性：** 行业分类来自 iFinD，部分股票可能缺失三级分类
3. **网络依赖：** 爬虫需要访问巨潮资讯网和 iFinD API，需保持网络畅通
4. **定时任务：** 依赖 Windows 任务计划程序，需保持电脑开机或配置唤醒
